# Eclipse Che Roadmap 🔥

# Short Term ⏱️

Epics with an estimated due date of 3 months or less

  • DevWorkspace engine MVP
  • Introduce a Projector-based IDEA Che editor
  • [Cross-Project Epic] Migrate rhel.Dockerfiles in all che repos to use ubi8 + install go-toolset RPMs (go-toolset container deprecated as of RHEL 8.4)
  • Add resource limits/requests to the devfiles and plugins in the registries
  • Jupyter Notebooks native support through latest Python VS Code extension
  • Create a workspace from a URL to a git repository in GitLab using a factory link
  • Create a workspace from a URL to a git repository in Bitbucket using a factory link
  • Expand support of devfile v2 in Che to include definition of Editors and Theia Plugins
  • Redesign Get Started page & Introduce Import from git
  • Update che-theia internal plug-ins to use new Devfile 2.0 compliant API
  • Multi-root as a default project layout in Che-Theia for simpler multi projects workspaces
  • [Cross-Project Epic] CI Build / Weekly Release process should use consistent quay tags for sha-version/next and 7.yy.z/latest
  • Create new v2 devfiles for current Che Getting Started samples

# Mid Term ⏰

Epics with an estimated due date between 3 and 12 months

  • Kubernetes Image Puller (KIP) should get its list of images to pull from the Che operator's CSV's RELATED_IMAGE_*
  • Deprecate installation of upstream Che using the OperatorHub
  • Ability to set resource quotas per users
  • Dashboard backend
  • Allow building and running applications at workspace start with devfile v2 post start events
  • Enable Che to more easily support latest runtime containers images through automation
  • Enable Che to more easily support latest plugin versions through automation
  • Che to perform git clone of projects to reduce errors and for easier integration with editors
  • Minimal Safari Support
  • Simpler user experience for working with terminals with an ‘all-in-one’ approach
  • Adopt DevWorkspace - a simpler and more lightweight kubernetes-native architecture, and the Devfile v2 specification for increased interoperability
  • Provide full lifecycle capabilities including creating and restoring backups in the Che operator
  • Synchronize changes between local devfile and workspace configuration in Che-Theia editor
  • Guide creation of devfile when not present, for easier persistence and sharing of workspace definitions
  • Automate trusted public certificates import in language stacks
  • Simplify authentication and authorization with a more flexible and lightweight approach
  • Enable community devfiles and plugins in Dev Sandbox

# Long Term ⏳

Epics with an estimated due date of 12 months or more

  • Support Multi-Arch Che (amd64, arm64, s390x, ppc64le)
  • Globalization Framework for Language Support
  • Make it possible to run multiple wsmaster in parallel
  • Live collaboration capabilities
  • Faster workspace startup experience
  • Use plug-ins of extension packs for all default stacks
  • Deprecate non-TLS deployments
  • Full support for Devfile 2.x spec
  • Multi cluster support
